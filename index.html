奇门三十六字真诀注释——揭露奇门最高机密

薛邓林（麟野山人）

此三十六决，之所以说是奇门遁甲之最高机密，乃是奇门用法之上境，且历代口传，甚少载之于书，即有亦是断简残篇，极难得见。非有缘者，纵阅历典籍千万，亦未能得此一二。有缘者当珍之、重之，切勿窜改妄传！

奇门三十六字真诀：

河洛连窍 先天穿宫 三局易气

六甲守门 八宫金锁 临制九变

遁甲归符 拆虚分象 浑天星仪

此决三十六字，已经将奇门遁甲的诸多秘法、功用、技巧泄露无遗。学者若能明了其中之奥，则奇门遁甲之精髓，已能尽得矣！此九句三十六字当是口传之决，基本未载于书，但秘本亦有类似之文记载，由此可窥书、决之同异：

河洛■合■■穷，动静顺逆窍相通。

对■摩荡终始气，洞穿先天幽冥机。

进兮退兮三卦详，九宫通气是一端。

八卦甲子神鬼藏，人门玄机两相参。

神机制敌金锁秘，？门捭阖泄天机。

生死存亡觉经纶，临制决机通九变。

■书行藏遁甲奥，万金莫与归符窍。

阴阳莫测天■象，■■分拆得玉函。

天■■气一掌间，浑天星■列乾坤。

其中■的地方为残缺，？处似为“人”字，又似“八”字，难以辨认。

奇门三十六字真诀当是从上文中凝炼而成，简明扼要，已经将奇门遁甲兵家、术家、道家运用之密包罗其中。传者有言：“不明此决，则不能尽得奇门遁甲之奥”，并不为过也。

此决乃奇门遁甲秘中之秘，到目前为之，世传的奇门，在运用方面仅得其学之一二，可见古人先贤藏决之深。本来不想对此注释，但目睹今日奇门研究之混乱，歪曲了奇门研究的重心与方向，思虑再三，尤恐不屑之辈得决后改头换面，以“道家”“真传”等面目伎俩惑世，故今略作诠注，学者若是有德有缘，自能悟出其中用法。

河洛连窍——河图：天一生水，地六成之；地二生火，天七成之；天三生木，地八成之；地四生金，天九成之；天五生土，地十成之。一、六共宗，二、七同道，三、八为朋，四、九为友，五、十同途。配于天干，则甲己合为一、六共宗，乙庚合为二、七同道，丙辛合为三、八为朋，丁壬合为四、九为友，戊癸合为五、十同途。天地生成，十干交合，则河图五行之体，化为十干之用。

洛书：戴九履一，左三右七，二四为肩，六八为足，五居其中。阳居四正，阴归四隅，中五居中，调停阴阳。上应北斗，下御八方。消长变化，尽在其中。

河图与洛书，一体一用，一常一变，一合一分。河图顺生：北方一六水生东方三八木，东方三八木生南方二七火，南方二七火生中央五十土，中央五十土生西方四九金，西方四九金生北方一六水。洛书逆克：坎、乾一六水克坤、兑二七火，坤、兑二七火克巽、离四九金，巽、离四九金克震、艮三八木，震、艮三八木克中五土，中五土克乾、坎一六水。

河图为先天，洛书为后天。先天者，人受制于天也；后天者，人能识天也，能识天，则能逆天。先天相生而发，后天相生而亡。若能于后天之逆中，反推先天之顺，则能体用兼得，逆而复顺，逆而得顺，以后天之克，成先天之生，此上一层作用也。

此乃河洛之妙窍，若能明此，则能知河洛之交媾，奇门遁甲九宫八门之窍，连而相通也。

先天穿宫——刚柔相摩，八卦相荡。能识河洛顺逆之用，则先后天之变化、对待、交媾，则能了然于心。先天八卦主气、运、时、数，后天八卦主方位空间。气运时数游离天地之间，无始无终，而北斗承气，运化四时，斡旋八方，天地相交，万物生化，有始有终，循环往来，四时而生，昼夜而成。精气为物，游魂为变。此亦道家养生之法。奇门遁甲，从后天之终，可察先天之始，后天之静，可窥先天之动，后天之常，可明先天之变。此为奇门遁甲“穿宫”之用。此穿宫，绝非世俗之“转宫”之法。而是从一宫之体、物中知其始末，详查先天之终始运行变化，从一宫到九宫（注：此一、九为数目，非坎一离九）中去察其来路，观其运行，若能明了来去，则能巧加运筹，趋生避死，趋旺避衰。用于占验，则能彰往察来。穿者，归于先天，将后天宫位相连也。

三局易气——奇门遁甲72局，乃效仿天地自然，以河洛、八卦九宫之数理而定，严密合于自然之气数，绝对妄作臆创。其局数之演变，其中有重大玄机。学者若能明此，则可悟奇门遁甲之重大秘密。这里提醒读者，注意节气、卦气，阴阳升降进退的变化。此本于天地河洛之数，乃天地交媾，万物化生之所在。而气之消长往来，有其持续性和规律性。知其首，则可明其尾，明其末，则可识其始也。假令冬至阳遁一局，天地二气玄关相交，中宫之戊统气于坎，坎之一，乃由大雪一之阴终，复由冬至一之阳始，再向七之阳演进，复到四之阳终，继而再由二之阳始。奇门遁甲绝非只是象数时空之式，更有气之流行作用存焉，而此乃奇门遁甲重中之重，惜为今世学者所忽视！若能知三局气之变易往来，天地交媾化生之通窍，则始能详查八方九宫之气，趋生避死矣！

六甲守门——六甲者，60甲子也，门者，八门也。今世之八门飞干飞支，乃其曲解误用也。六甲隐于地盘三奇六仪之下，遁甲之法，以天地二盘之奇仪体察六甲干支，作为天地二气之消长变化。奇仪者，天体之气也。六甲者，时序之气也。人（人盘）载天覆地，于天盘之下，地盘之上，从天地相交之气中，可以知八方之生死衰旺。八门者，人事吉凶之用也，然奇门遁甲，九宫八方之气一定，但生死存亡则取决于人之动变。生门非生，死门非死，善趋避者，能将从天地六甲之气运中，妙布八门，则能扭转乾坤。六甲统于六仪，甲子遁于奇仪之下，若能察其时序，识其隐显，巧识九宫之遁，巧统八方之摄，则虽失三奇三吉之利，亦能巧用六甲，巡守八门，则吾死非死，彼生非生也！

八宫金锁——此亦有称为“八门”金锁者，门与宫，虽名有异，而其用法，则一也。乃将八门游变八方，门锁，亦即是宫锁。若论式盘，时家奇门八门布于九宫，一时一易。依活盘法式，若生居东，则死必居西；若开居南，则杜必居北也。此为奇门占测之板法。但奇门筹划，则有活变之法，此乃兵家奇门之秘，术者不识。亦即兵家阵法之秘，《三国演义》所谓“八门阴阳，反复变迁”即是此意，抄本《天机素书》略露片语，此法乃配于六甲使用。八阵之用，变化无常，虽奇门式盘生死等门方位一定，但善用八门之变者，则能详敌意，彼何方进攻，我则能轮转生死，御其进攻也。注：此非刻家奇门之谓。乃据式盘之板法而变，其法，略露一句：“入式八门轮八方，临阵八门布九宫。”八门演布九宫之法，乃据天道之常，寓人道之变也。此方亦能在奇门占测中用于“锁”定某些特殊的事物，从八门之变中窥其吉凶。

临制九变——此为奇门遁甲之极高层运用。奇门遁甲之九宫八卦，看似各宫之间各自为政，孤立为一，实则是息息相通，为一个相互联系不可分割的整体。而一宫之中，藏有九种变化。亦即九宫是一个大太极，而每卦则是一个小太极，每一宫之中，都能控制玄关，调停阴阳。若能察时令、气数之变易，详天关地窍之所在，则能运用其中一处，临制九变，控制八方。此句和下两句，乃是奇门遁甲重中之重，为防鼠辈盗用窜改以惑世，今只在此略露端倪，但相信和奇门有缘的读者能够从中领会。

奇门以宫配卦，以卦定局，有其重大玄机存焉。如一宫配坎卦，统冬至、小寒、大寒三节气，亦即阳遁1至9局全备，此乃一宫妙含九局，一卦统摄八方，若能明此深层用法，则可以一卦得九局之变。决曰：“一卦而得九，中宫八方有。”

拆虚分象——决曰：“六甲有旬空，八方有孤虚”，学者但知四建之旬空，未识六甲之虚空。奇门遁甲，将60甲子分布九宫，合天地两盘，交而用之，六仪统气之所，符首管辖之地，有其虚空深意存也。天地于交媾之中，自有盈虚，此为鬼神莫测之变，乃是实中求虚，静中观动最上一层的用法。晓此之用者，则能妙击左右，斩关破阵，辩四方之真虚实也。此与上二，传决者只可心领意会，皆未敢直白相告，非德者，不能窥此天地之密也。

##white-space
white-space:nowrap
永不换行，只有
存在时才换行

white-space:pre：
空格和换行符全都被保留了下来！不过自动换行还是没了。保留，所以pre其实是preserve的缩写，这样就好记了。

white-space:pre-wrap：
显然pre-wrap就是preserve+wrap，保留空格和换行符，且可以自动换行。

white-space:pre-line：
空格被合并了，但是换行符可以发挥作用，line应该是new line的意思，自动换行还在，所以pre-line其实是preservenew line+wrap。

##word-break
word-break:keep-all：
所有“单词”一律不拆分换行，注意，我这里的“单词”包括连续的中文字符（还有日文、韩文等），或者可以理解为只有空格可以触发自动换行

word-break:break-all：
所有单词碰到边界一律拆分换行，不管你是incomprehensibilities这样一行都显示不下的单词，还是long这样很短的单词，只要碰到边界，都会被强制拆分换行。所以用word-break:break-all时要慎重呀。

CSS加载会造成阻塞吗？

发表于 2020-01-13 | 分类于 CSS

#CSS加载会造成阻塞吗？
1、css加载不会阻塞dom树的解析
2、css加载会阻塞dom树的渲染
3、css加载会阻塞后面js语句的执行

在一个html文件中，js放在前面，css放在后面，可以很好的避免css阻塞状况的发生。css语句后面不能再放js语句，否则改语句就会等到css执行完成后才执行。

#如何提高css加载速度？
1、使用CDN(因为CDN会根据你的网络状况，替你挑选最近的一个具有缓存内容的节点为你提供资源，因此可以减少加载时间)

2、对css进行压缩(可以用很多打包工具，比如webpack,gulp等，也可以通过开启gzip压缩)

3、合理的使用缓存(设置cache-control,expires,以及E-tag都是不错的，不过要注意一个问题，就是文件更新后，你要避免缓存而带来的影响。其中一个解决防范是在文件名字后面加一个版本号)

4、减少http请求数，将多个css文件合并，或者是干脆直接写成内联样式(内联样式的一个缺点就是不能缓存)

#浏览器渲染方式？
主要有webkit渲染过程和gecko渲染过程

渲染流程如下

1、HTML解析文件，生成DOM tree，解析css文件生成CSSOM Tree
2、将DOM Tree和CSSOM Tree结合，生成Render Tree（渲染树）
3、根据Render Tree渲染绘制，将像素渲染到屏幕上

从流程我们可以看出来

1、DOM解析和CSS解析是两个并行的进程，所以这也解释了为什么CSS加载不会阻塞DOM的解析。

2、然而，由于Render Tree是依赖于DOM Tree和CSSOM Tree的，所以他必须等待到CSSOM Tree构建完成，也就是CSS资源加载完成(或者CSS资源加载失败)后，才能开始渲染。因此，CSS加载是会阻塞Dom的渲染的。

3、由于js可能会操作之前的Dom节点和css样式，因此浏览器会维持html中css和js的顺序。因此，样式表会在后面的js执行前先加载执行完毕。所以css会阻塞后面js的执行。

#DOMContentLoaded
对于浏览器来说，页面加载主要有两个事件，一个是DOMContentLoaded，另一个是onLoad。而onLoad没什么好说的，就是等待页面的所有资源都加载完成才会触发，这些资源包括css、js、图片视频等。

而DOMContentLoaded，顾名思义，就是当页面的内容解析完成后，则触发该事件。

我们可以看到，只有在css加载完成后，才会触发DOMContentLoaded事件。因此，我们可以得出结论：

1、如果页面中同时存在css和js，并且存在js在css后面，则DOMContentLoaded事件会在css加载完后才执行。

2、其他情况下，DOMContentLoaded都不会等待css加载，并且DOMContentLoaded事件也不会等待图片、视频等其他资源加载。

#for/for in/for of/forEach的区别
1、for循环在Array和Object中都可以使用

var arr= [1,4,6,5,2,3,4,5];
for（var i=0;i<arr.length;i++）{
console.log(arr[i]);
}
//1,4,6,5,2,3,4,5
//普通for循环在对象中不实用，因为属性的命名和key的值会存在各种限制，不适合用在对象中
var obj1 = {
    x0:100,
    x1:200,
    x3:300
}
for(var key = 0;key < 3;key++){
console.log(obj1["x"+key]);//这里需要指定键名才能输出值名；
//100,200,300

2、for in在Array和Object中都可以使用

在对象上还包含了原型上的属性。

var arr= [1,4,6,5,2,3,4,5];
for(var i in arr){
console.log(arr[i])
if(arr.hasOwnProperty(i)){
    console.log(arr[i])
    }
}
var obj = {
       `x:100,`
    y:200,
    z:300
}
obj.__proto__.a=400;
for(var key in obj){
    console.log(obj[key]);
    if(obj.hasOwnProperty(key)){
        console.log(obj[key]);
    }
}

3、for of在Array，Object，Set,Map中都可以使用

var arr= [1,4,6,5,2,3,4,5,90];
for(let index of arr.keys()){
console.log(index)
}
//0,1,2,3,4,5,6,7,8
for(let value of arr.values()){
console.log(value)
}
//报错
for(let value of arr){
console.log(value)
}
//1,4,6,5,2,3,4,5,90
for(let [index,value] of arr.entries()){
console.log(index,value)
}

var obj = {
x:100,
y:200,
z:300
}

for(let [key,value] of Object.entries(obj)){
console.log(key,value);
}

let arr = [“add”,”delete”,”clear”,”has”];
let list = new Set(arr);
for(let key of list.keys()){
console.log(“key”,key);
}

for(let value of list.values()){
console.log(“value”,value);
}

for(let [key,value] of list.entries()){
console.log(“value”,key,value);
}

let map = new Map();

console.log(map);

for(let key of map.keys()){
console.log(“value”,value);
}

for(let [key,value] of map.entries()){
console.log(“entries”,key,value);
}

list.forEach((value,key)=>{
console.log(“forEach”,key,value);
